 🔄 FLUXO DE AUTENTICAÇÃO
══════════════════════════════════════════════════════════════════════

  FRONTEND (Login.tsx)
  ├─ Detecta: VITE_DEMO_MODE
  │  ├─ true  → usa credenciais do .env.local (desenvolvimento)
  │  └─ false → chama POST /api/auth/login (produção)
  │
  └─ Armazena em localStorage
     ├─ auth_token: "eyJh..." (JWT token)
     └─ lanchonete_session: {id, name, role, token}

  BACKEND (auth.js + middleware)
  ├─ POST /api/auth/login
  │  └─ Valida credenciais → retorna JWT token
  │
  ├─ POST /api/auth/validate
  │  └─ Verifica se token é válido
  │
  ├─ authMiddleware
  │  └─ Protege rotas (valida token em cada requisição)
  │
  └─ roleMiddleware
     └─ Verifica role do usuário (admin, operador, caixa)


 🚀 COMO TESTAR AGORA
══════════════════════════════════════════════════════════════════════

  1️⃣  DESENVOLVIMENTO (Modo Demo)
      $ cd frontend && npm run dev
      → http://localhost:5173
      → Credenciais: admin / admin123
      → Não precisa de backend rodando

  2️⃣  PRODUÇÃO (API Real)
      Terminal 1: $ cd backend && npm start
      Terminal 2: $ cd frontend && npm run dev
      → Mude VITE_DEMO_MODE=false em .env.local
      → Login enviará POST para /api/auth/login

  3️⃣  COM CURL (Teste Manual)
      $ curl -X POST http://localhost:3000/api/auth/login \
          -H "Content-Type: application/json" \
          -d '{"username":"admin","password":"admin123"}'
      → Retorna token JWT
