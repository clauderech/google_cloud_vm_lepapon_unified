🎓 CONCEITOS APRENDIDOS
═══════════════════════════════════════════════════════════════════════

JWT (JSON Web Token)
  • Token stateless com informações do usuário
  • Estrutura: header.payload.signature
  • Assinado com JWT_SECRET
  • Expira automaticamente (7 dias por padrão)

Middleware de Autenticação
  • Função que valida token antes de executar rota
  • Adiciona req.user com dados do usuário
  • Suporta verificação de role/permissão

Modo Demo
  • Ideal para desenvolvimento local
  • Valida contra variáveis de ambiente
  • Não precisa de backend rodando
  • Fácil de ativar/desativar

Segurança
  • Credenciais em variáveis de ambiente
  • .env.local é privado (git-ignored)
  • Token usado em requisições futuras
  • Validação automática em cada acesso


📊 DIAGRAMA DO SISTEMA
═══════════════════════════════════════════════════════════════════════

Frontend (React)
    ↓
Login.tsx detecta VITE_DEMO_MODE
    ├─ Se true: Valida localmente (demo)
    └─ Se false: POST /api/auth/login (produção)
        ↓
Backend (Express)
    ├─ /api/auth/login
    │   └─ Valida credenciais → JWT.sign()
    ├─ /api/auth/validate
    │   └─ JWT.verify() → autorizar
    ├─ /api/auth/logout
    │   └─ Cleanup token
    └─ /api/auth/me
        └─ Retorna dados do usuário

Token armazenado em localStorage
    ↓
Requisições futuras com Authorization header
    ↓
authMiddleware valida token
    ↓
Rota protegida executa com req.user
